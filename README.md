## 📝 Описание проекта
Данный проект представляет собой backend-часть веб-платформы для самообучения. Платформа позволяет администраторам и преподавателям создавать структурированные курсы с уроками и тестами, а студентам — изучать материалы и проверять свои знания.

Проект реализован на **Django** и **Django REST Framework (DRF)** с использованием **PostgreSQL** в качестве базы данных. Аутентификация построена на основе **JWT (JSON Web Tokens)**, что обеспечивает безопасный и stateless доступ к API.

## 🚀 Функциональные возможности
Проект разделен на три ключевых приложения, каждое из которых отвечает за свой функционал:

### 1. 👥 Управление пользователями (users)
* Регистрация новых пользователей с ролью "Студент".

* Аутентификация по email/username и паролю с выдачей JWT-токенов (access и refresh).

* Разграничение прав доступа на основе ролей: **Администратор, Преподаватель, Студент**.

* Администраторы могут просматривать всех пользователей, а также редактировать их роли и данные.

* Пользователи могут просматривать только свой профиль.

### 2. 📚 Управление курсами (courses)
* **Курсы (Course):** Создание, просмотр, обновление и удаление учебных курсов.

* **Уроки (Lesson):** Наполнение курсов уроками с текстовым описанием, превью и ссылками на видео.

* Права доступа:

    * **Преподаватели:** Могут создавать и управлять только своими курсами и уроками.

    * **Администраторы:** Полный контроль над всеми курсами и уроками.

    * **Студенты:** Только просмотр материалов.

### 3. 📝 Тестирование знаний (tests)
* **Тесты (Test):** Привязка тестов к урокам (связь "один к одному"). Настройка лимита времени, количества попыток и проходного балла.

* **Вопросы (Question):** Создание вопросов с выбором одного или нескольких правильных ответов.

* **Варианты ответов (AnswerOption):** Добавление вариантов с отметкой о правильности.

* **Прохождение тестов:**

    * Студенты могут начать попытку (start_attempt) и отправить ответы (submit).

    * Сервер автоматически проверяет ответы, подсчитывает баллы и фиксирует результат.

    * Ведется история попыток (TestAttempt) и результатов (TestResult).

* Права доступа:

    * **Преподаватели:** Управляют тестами только для своих уроков.

    * **Студенты:** Проходят только активные тесты и видят свои результаты.

    * **Администраторы:** Полный доступ.

## 🛠 Технологический стек

* **Язык:** Python 3.13+

* **Фреймворк:** Django 5.2, Django REST Framework 3.16

* **База данных:** PostgreSQL

* **Аутентификация:** JWT (djangorestframework-simplejwt)

* **Документация API:** Swagger (drf-yasg)

* **CORS:** django-cors-headers

* **Статика:** WhiteNoise

* **Инструменты разработки:**

    * black, isort, flake8, mypy — для форматирования и проверки качества кода.

    * coverage — для измерения покрытия кода тестами.

    * python-dotenv — для управления переменными окружения.

## ⚙️ Установка и запуск проекта

Следуйте этим шагам, чтобы запустить проект локально.

### Предварительные требования
* Установленный **Python** версии 3.13 или выше.

* Установленный **Poetry** для управления зависимостями: pip install poetry

* Установленный и запущенный **PostgreSQL**.

### Шаг 1: Клонирование репозитория

git clone https://github.com/Alexandra03091986/self_learning_platform/tree/feature/backend

### Шаг 2: Установка зависимостей
Активируйте виртуальное окружение Poetry и установите все необходимые пакеты, включая зависимости для разработки:

```
poetry shell
poetry install 
```

### Шаг 3: Настройка переменных окружения
Создайте файл .env в корне проекта и скопируйте в него содержимое из .env.example.
Заполните его своими данными.
```
# Django
SECRET_KEY=your-super-secret-key-here-change-in-production
DEBUG=True

# Database
POSTGRES_DB=self_learning_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_db_password
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
```

### Шаг 4: Создание базы данных
В PostgreSQL создайте базу данных с именем, указанным в вашем .env файле (например, self_learning_db).

### Шаг 5: Применение миграций
Выполните миграции для создания всех таблиц в базе данных:

```
python manage.py migrate
```

### Шаг 6: Создание суперпользователя
Создайте администратора для доступа в Django admin:

```
python manage.py createsuperuser
```
### Шаг 7: Сбор статических файлов
```
python manage.py collectstatic
```
### Шаг 8: Запуск сервера разработки
```
python manage.py runserver
```

## 🧪 Тестирование
В проекте написаны тесты для всех ключевых функций. Для их запуска выполните команду:

```
python manage.py test
```

Для проверки покрытия кода тестами используйте:
```
coverage run --source='.' manage.py test
coverage report
# или для просмотра в HTML формате
coverage html
# затем откройте htmlcov/index.html в браузере
```
Текущий уровень покрытия кода тестами составляет 96%, что свидетельствует о высокой надежности проекта.
Открыв [index_htmlcov.html](index_htmlcov.html) можно увидеть отчет о покрытии проекта.

## 👥 Роли и права доступа
* 👑 **Администратор** (admin)

    * Полный доступ ко  всем функциям платформы.

    * Управление пользователями (создание, редактирование, удаление, изменение ролей)
    * Просмотр и редактирование любых курсов, уроков и тестов

    * Доступ в Django admin (/admin/).

* 👨‍🏫 **Преподаватель** (teacher)

    * Создание, редактирование и удаление своих курсов и уроков

    * Создание и управление тестами для своих уроков

    * Просмотр результатов студентов по своим курсам

    * Не может изменять чужие материалы

* 👩‍🎓 **Студент** (student)

    * Просмотр всех доступных курсов и уроков

    * Прохождение активных тестов

    * Просмотр своих результатов и истории попыток

    * Редактирование только своего профиля
